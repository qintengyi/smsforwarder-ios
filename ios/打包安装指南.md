# SmsForwarder iOS — 打包与全能签安装指南

本指南把 **iOS 源码工程** 用 **GitHub 云端 macOS** 编译成未签名 `SmsForwarder.ipa`，再用 **全能签** 在 iPhone 16 Pro (iOS 18.6) 上重签名安装。全程无需 Mac、无需苹果开发者账号付费。

---

## 0. 你需要准备的东西

| 项目 | 说明 |
|------|------|
| GitHub 账号 | 免费，注册 github.com |
| 全能签 | 已安装在你的 iPhone 16 Pro 上 |
| p12 证书 + .mobileprovision 描述文件 | 你已有；**描述文件必须包含这台 iPhone 16 Pro 的 UDID**，否则装不上 |
| 本工程的 `ios/` 目录 + `.github/` 目录 | 即本仓库内容 |

> ⚠️ 关键：描述文件（.mobileprovision）里的设备 UDID 必须包含你要安装的那台 iPhone 16 Pro。全能签重签时会校验，UDID 不匹配则安装失败。

---

## 1. 上传工程到 GitHub

### 1.1 新建仓库

1. 登录 GitHub → 右上角 **+** → **New repository**
2. Repository name：`smsforwarder-ios`
3. 选 **Public**（公开仓库 GitHub Actions 免分钟数；私有仓库每月也有 2000 分钟，够用）
4. **不要**勾选 "Add a README"、".gitignore"、"license"（保持空）
5. 点 **Create repository**

### 1.2 上传文件

仓库根目录结构应该是这样：

```
smsforwarder-ios/
├── ios/                      ← 工程源码
│   ├── project.yml           ← XcodeGen 配置
│   ├── Sources/SmsForwarder/ ← 全部 Swift 源码
│   └── ...
└── .github/
    └── workflows/
        └── build-ipa.yml     ← 自动打包脚本
```

**方式 A：网页上传（最简单，不用装 git）**

1. 在刚建好的空仓库页面点 **uploading an existing file**
2. 把本仓库的 `ios/` 文件夹 和 `.github/` 文件夹 拖进去
   - 注意：网页上传会丢失空文件夹，但本工程没有依赖空文件夹，可放心
   - `ios/` 里所有子目录文件都要传上去（Sources、project.yml）
   - `.github/workflows/build-ipa.yml` 一定要传
3. 填 commit message，点 **Commit changes**

> 提示：网页上传嵌套文件夹时，浏览器有时会把所有文件摊平。如果发现层级乱了，改用下面的命令行方式最稳。

**方式 B：命令行上传（推荐，结构最准）**

```bash
# 在本工程根目录打开终端
git init
git remote add origin https://github.com/你的用户名/smsforwarder-ios.git
git branch -M main
git add ios .github
git commit -m "feat: SmsForwarder iOS 工程与 GitHub Actions 打包"
git push -u origin main
```

> 如果本目录已是 git 仓库，只需 `git push` 到新 remote 即可。

## 2. 触发云端编译

代码推上去后，GitHub Actions 会**自动触发**编译（push 到 main/master 即触发）。

1. 打开仓库页面 → 顶部 **Actions** 标签
2. 左侧选 **Build IPA** → 可看到一个正在跑的 workflow（黄圈=进行中，绿勾=成功，红叉=失败）
3. 等 5~10 分钟，看到绿勾即编译成功
4. 点进这次成功的运行 → 页面下方 **Artifacts** 区有 `SmsForwarder-unsigned-ipa` → 点它下载
5. 下载到的是一个 zip，解压后得到 **`SmsForwarder.ipa`**

> 如果红叉失败：点进运行看红色那一步的日志，把报错信息发给我，我帮你修。
> 也可以手动触发：Actions → Build IPA → 右侧 **Run workflow** →绿色按钮。

## 3. 全能签重签并安装到 iPhone 16 Pro

### 3.1 导入证书到全能签

1. 把你的 `xxx.p12` 和 `xxx.mobileprovision` 传到 iPhone（AirDrop / 微信 / 邮件均可）
2. 打开**全能签** → 我的/设置 → 证书管理
3. 导入 p12（会让你输 p12 密码）→ 再导入 .mobileprovision
4. 确保证书状态显示"有效"

### 3.2 导入并重签 ipa

1. 把第 2 步下载的 `SmsForwarder.ipa` 传到 iPhone
2. 在全能签里选择"导入 ipa"或直接打开这个 ipa 文件 → 选择用全能签打开
3. 全能签会自动用你导入的证书**重签名**
4. 重签完成 → 点**安装**
5. 桌面出现 SmsForwarder 图标

### 3.3 信任开发者证书（很重要，不信任会闪退）

iOS 18.6 安装第三方签名 App 后必须手动信任：

1. 设置 → 通用 → VPN与设备管理（或"描述文件与设备管理"）
2. 找到你的开发者证书 → 点进去 → **信任**
3. 回桌面点 SmsForwarder 图标即可打开

> 如果点开闪退，99% 是没信任证书，或者描述文件的 UDID 不包含这台手机。

## 4. 首次使用 App

1. 打开 App → 底部 **更多** Tab → **设置**
2. 填写：
   - 设备 IP：你的 SmsForwarder 手机局域网 IP（默认 192.168.1.16）
   - 端口：5000
   - Secret Key：与服务器端一致的密钥
3. 点**测试连接**，提示成功即可
4. 回**仪表盘**下拉刷新，看到设备在线就大功告成 ✅

---

## 5. 重新打包（以后改了代码）

改完源码后只要 `git push`，Actions 自动重新编译出新 ipa，下载安装即可。无需任何手动命令。

## 6. 常见问题

**Q：GitHub Actions 显示失败怎么办？**
A：点 Actions → 失败的那次 → 看红色步骤日志，多半是文件没传全。确保 `ios/Sources/SmsForwarder/` 下所有 .swift 和 `ios/project.yml`、`.github/workflows/build-ipa.yml` 都已上传。

**Q：全能签提示"SHSH"或安装失败？**
A：
- 确认 .mobileprovision 里有这台手机的 UDID（去证书商那里把手机 UDID 加进去重新下载描述文件）
- iOS 18.6 建议用全能签最新版
- p12 和描述文件要配套（来自同一开发者）

**Q：App 打开就闪退？**
A：去 设置 → 通用 → VPN与设备管理 → 信任你的开发者证书。

**Q：连接设备提示"无法连接"？**
A：确认 iPhone 与 SmsForwarder 那台手机在同一 WiFi 局域网；IP/端口/密钥填对。

**Q：能改成已签名 ipa 直接装吗（不用全能签重签）？**
A：可以，需要在 GitHub 仓库 Settings → Secrets 配置 `P12_BASE64`、`P12_PASSWORD`、`PROVISION_BASE64`，再修改 workflow 用 xcodebuild archive 签名打包。未签名 + 全能签重签最省事，本仓库默认用此方式。
